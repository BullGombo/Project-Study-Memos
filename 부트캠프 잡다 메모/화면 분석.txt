화면명 : 대시보드

1) 표시되는 데이터

----- <오늘 기준 날짜>
	(오늘 날짜 - 날짜형 etc)


----- <개요>
- [전체 작업] 
	(갯수 count - 정수형)
- [완료] 
	(갯수 count - 정수형)
- [진행 중] 
	(갯수 count - 정수형)
- [기한 초과]
	(갯수 count - 정수형)
- [내 작업]
	(각 작업 제목/작성자명 - 작업 객체형 -> 문자형 추출)
- [활동 피드]
	(각 로그 메세지/날짜 - 로그 객체형  -> 문자형/날짜형 추출)


----- <통계 및 차트>
- [작업 상태]
	(나의 모든 작업의 할일/진행 중/완료 갯수 count - 정수형)
			! 기한 만료된 작업은 미포함
			
- [진행률]
	(로그인한 계정의 작업/팀의 작업 갯수와 완료 상태 갯수 count - 정수형)
			따라서 실제 넘겨주는 값은 => 
				내 완료 작업/전체 작업 백분율, 팀 완료 작업/팀 전체 작업 백분율
				
- [주간 작업 추세]
	(오늘 포함 +-3일의 날짜별 총 작업/완료된 작업 갯수 count - 정수형) 
	혹은 
	(총 작업 시작일과 종료일 + 해당일의 작업 갯수)


2) 입력 폼
-- 입력폼 없음


3) 버튼/액션
- [통계 및 차트] 버튼 → 통계 및 차트 화면 표시
- [개요] 버튼 → 개요 화면 표시


4) 특이사항/비즈니스 룰 (비고)
- 해당 화면은 입력(request) 폼이 존재하지 않음
- <개요>/<통계 및 차트> 표시 여부와 상관없이 오늘 기준의 날짜 데이터를 필요로 함(상단 표시, 주간 작업 추세의 +-3일 기준용)
- 해당 페이지는 <개요>/<통계 및 차트> 표시 여부와 상관없이 한번에 모든 데이터를 가져옴
