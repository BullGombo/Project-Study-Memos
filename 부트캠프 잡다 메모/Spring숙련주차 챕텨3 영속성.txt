📌트랜잭션 전파
Spring에서는 @Transactional 어노테이션을 사용해 메서드를 트랜잭션 안에서 실행하게 할 수 있음
이때 propagation 옵션으로 여러 가지 전략을 설정이 가능

1. REQUIRED (default)
2. REQUIRES_NEW
3. NESTED
4. SUPPORTS
5. NOT_SUPPORTED
6. MANDATORY
7. NEVER

MANDATORY와 NEVER 옵션
MANDATORY와 NEVER는 트랜잭션 옵션이라기보다는 안전 장치 역할.

실제로 사용하는 것은 REQUIRED가 99.9%

✅ 트랜잭션 격리 수준 (Transaction Isolation Level)

트랜잭션 격리 수준은 격리(고립) 수준이 낮은 순서대로 READ UNCOMMITED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE가 존재
READ UNCOMMITED
READ COMMITTED
REPEATABLE READ
SERIALIZABLE
 
💬 NON-REPEATABLE READ와 PHANTOM READ의 차이
NON-REPEATABLE READ는 "기존 데이터의 값 변경" 문제이고, PHANTOM READ는 "데이터 행의 추가/삭제" 문제

📌연관관계
연관관계는 엔티티(테이블) 간의 관계를 의미

📌Primary Key와 Foreign Key

데이터베이스에서 테이블 간의 관계를 설정하고 데이터의 무결성을 보장하는 가장 핵심적인 두 요소는 기본 키(Primary Key, PK)와 외래 키(Foreign Key, FK)

특징 : 유일성, NOT NULL, 불변성

Foreign Key
외래 키는 한 테이블의 컬럼이 다른 테이블의 기본 키를 참조하는 것을 말합니다. 즉, 테이블 간의 관계를 맺어주는 연결 고리 역할

특징 : 관계 설정, 중복 허용, NULL 허용

고아 제거(orphanRemoval) 학습

 ✅ 영속성 전이 (Cascade)

1. CascadeType.ALL
모든 Cascade 옵션을 적용합니다. (PERSIST + REMOVE + MERGE + REFRESH + DETACH)

2. CascadeType.PERSIST
설명:
부모 엔티티를 저장할 때, 연관된 자식 엔티티도 함께 저장됩니다.

3. CascadeType.REMOVE

4. CascadeType.MERGE

5. CascadeType.REFRESH

6. CascadeType.DETACH

💬Cascade 옵션은 언제 어떤 걸 써야하나요?
안 쓰거나, CascadeType.ALL을 쓰거나입니다.
일단 실무에서는 양방향 매핑을 사용하지 않으니 @OneToMany도 사용하지 않고, Cascade 옵션도 잘 사용하지 않게 됩니다.
실무자 입장에서 이러한 Cascade 옵션은 굉장히 위험할 수 있습니다. 왜냐하면 함부로 데이터 값을 변경하거나 삭제할 수 있기 때문입니다.
데이터는 곧 돈이기에 코드로 명시적으로 통제하는 경우가 대부분입니다.

✅ 고아 제거 (Orphan Removal)

orphanRemoval = true
@OneToMany(orphanRemoval = true) 혹은 @OneToMany(orphanRemoval = false) 형태로 사용

1. 부모의 컬렉션에서 자식을 제거할 때

2. 부모가 새로운 컬렉션으로 교체될 때

3. 부모 엔티티가 삭제될 때
