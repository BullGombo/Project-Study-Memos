아래는 UserService의 마이페이지를 조회하는 두가지 메서드
getMe()
getOtherUser(Long userId)
위 메서드에서 사용하는 응답객체 각각이 정확히 같은필드를 전달해서, 사실상 전달하는 필드가 같은데, 이부분을 기능상 혹은 dto 데이터상 차이를 두고싶었다

-> 서로 다른 DTO가 필요함
내 정보(getMe)와 타 유저 정보(getOtherUser)는 기능적으로는 완전히 다른 목적

| 조회 대상                                | 보안 요구 수준   | 보여줄 수 있는 정보 | 예시                    |
| ------------------------------------ | ---------- | ----------- | --------------------- |
| **내 정보 (GetUserResponse)**           | 가장 높은 권한   | 개인 정보 포함 가능 | 이메일 변경, 닉네임 변경, 활동 로그 |
| **다른 사용자 정보 (GetOtherUserResponse)** | 공개 가능한 정보만 | 개인정보 제한     | 이메일 숨김, 비공개 계정 표시     |

🧩 개선 방향 2 — 공통 데이터는 Base DTO로 분리
중복을 줄이면서 목적도 분리하고 싶다면:
@Getter
@RequiredArgsConstructor
public abstract class BaseUserResponse {
    private final long id;
    private final String nickname;
    private final String userName;
    private final LocalDateTime createdAt;
}

그리고 각 DTO가 확장한다:
@Getter
public class GetUserResponse extends BaseUserResponse {
    private final String email;
    private final LocalDateTime modifiedAt;
    private final boolean marketingConsent;

    public GetUserResponse(User user) {
        super(user.getId(), user.getNickname(), user.getUserName(), user.getCreatedAt());
        this.email = user.getEmail();
        this.modifiedAt = user.getModifiedAt();
        this.marketingConsent = user.isMarketingConsent();
    }
}
java
---
@Getter
public class GetOtherUserResponse extends BaseUserResponse {
    private final boolean isFollowing;
    private final int followerCount;

    public GetOtherUserResponse(User user, boolean isFollowing, int followerCount) {
        super(user.getId(), user.getNickname(), user.getUserName(), user.getCreatedAt());
        this.isFollowing = isFollowing;
        this.followerCount = followerCount;
    }
}


🧩 개선 방향 3 — View Projection 패턴(권한 기반 필터링)
권한이 다르면 반환되는 필드도 달라지는 방식
public interface UserProfileMapper {
    GetUserResponse toMyProfile(User user);
    GetOtherUserResponse toPublicProfile(User user);
}

이 방식은 나중에 Admin 조회 DTO, 친구 전용 DTO, 비공개 계정 DTO도 확장 가능
